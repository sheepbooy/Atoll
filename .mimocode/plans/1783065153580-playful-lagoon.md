## 一、安全问题（Security）

### 🔴 S1. CSP 被完全禁用
- **文件**: `src-tauri/tauri.conf.json:32`
- **问题**: `"csp": null` 完全关闭了内容安全策略，允许 webview 中执行任意脚本。若有用户可控内容被渲染（如 Markdown 消息），XSS 攻击面大幅增加。
- **修复**: 设置合理的 CSP，至少限制 `script-src 'self'`、`style-src 'self' 'unsafe-inline'`。

### 🟡 S2. `openUrl()` 未校验 URL 协议（复核：原 P0 描述不成立，建议防御性加固）
- **文件**: `src/tauri.ts:637-643`, `src/App.tsx:3915-3920`, `src/App.tsx:5041-5046`
- **原问题**: Markdown 中的链接点击直接调用 `openUrl(anchor.href)`，未过滤 `file://`、`javascript:` 等危险协议。
- **复核结果**: `openUrl()` 本身没有协议白名单属实；但当前链接来自 `ReactMarkdown`，其默认 `defaultUrlTransform` 只允许安全协议或相对路径。危险协议会被转换成空 href attribute，因此“可通过 Markdown 直接打开 `javascript:` / `file:`”这一描述不成立。需要注意的是，点击处理读取的是 `anchor.href` DOM property，空 href property 在浏览器里通常会解析成当前页面 URL，而不是原始空字符串。
- **修复建议**: 仍可在 `openUrl()` 或点击处理处增加 `http:` / `https:` 白名单作为 defense-in-depth，但优先级应降为 P3。若加白名单，优先使用 `anchor.getAttribute("href")` 或 `URL` parser 判断协议，并评估是否保留 `mailto:` 等当前被 `react-markdown` 允许的协议。

### 🟠 S3. `get_session_transcript` 路径遍历风险
- **文件**: `src-tauri/src/lib.rs:1209-1214`, `src-tauri/src/lib.rs:1392-1395`, `src-tauri/src/hook_bridge.rs:1525-1531`
- **问题**: 接受前端传入的任意文件路径，无沙箱校验。若前端被攻破，可读取进程权限范围内的任意文件。
- **修复**: 校验路径必须在预期的 session 目录内，拒绝包含 `..` 的路径。

### 🟠 S4. 本地 Hook Bridge 无认证
- **文件**: `src-tauri/src/hook_bridge.rs:24-25`, `src-tauri/src/hook_bridge.rs:570-578`
- **问题**: localhost:47777 的 HTTP 服务无任何认证、API Key 或 CORS 限制，本机任意进程均可发送请求。
- **修复**: 添加随机 token 认证或限制请求来源。

### ⚪ S5. IPC 命令写入用户配置文件缺乏输入校验（复核：原描述不准确）
- **文件**: `src-tauri/src/lib.rs:3318`, `src-tauri/src/lib.rs:3728`, `src-tauri/src/lib.rs:3931`
- **原问题**: 可直接写入 `~/.claude/settings.json`、`~/.codex/hooks.json` 等，未对输入做消毒。
- **复核结果**: `install_claude_hooks(app)`、`install_codex_hooks(app)`、`install_cursor_hooks(app)` 没有接收前端传入的任意配置 JSON，而是根据本地解析出的 node/script 路径写入固定 hook 模板。因此“不消毒前端输入导致任意配置写入”不成立。
- **保留风险**: 仍建议改进现有配置文件的 schema/结构校验，避免 malformed JSON 被静默重置为空对象，或 hook 合并时破坏用户已有配置。但这不是当前报告描述的安全输入校验问题。

### 🟡 S6. `.gitignore` 缺少 `.env` 等敏感文件规则
- **文件**: `.gitignore`
- **问题**: 缺少 `.env`、`.env.*`、`*.key`、`*.pem`、`coverage/` 等常见规则。
- **修复**: 补充以下规则：
  ```
  .env
  .env.*
  *.key
  *.pem
  coverage/
  ```

## 二、内存泄漏与资源清理（Memory Leaks & Cleanup）

### 🔴 M1. Tauri 事件监听器泄漏（所有 `listen()` 调用点）
- **文件**: `src/App.tsx:830-940`, `src/TokenCounter.tsx:263-290`
- **问题**: 所有 `on*Changed()` 返回 `Promise<UnlistenFn>`。若组件在 Promise resolve 前卸载，cleanup 函数为空 `() => {}`，真正的监听器永远不会被取消。这是一个系统性问题，影响约 8 个监听点。
- **修复**: 使用 `unmounted` 标志模式：
  ```tsx
  useEffect(() => {
    let unmounted = false;
    let cleanup: (() => void) | null = null;
    onSomeEvent(handler).then((fn) => {
      if (unmounted) fn();
      else cleanup = fn;
    });
    return () => {
      unmounted = true;
      cleanup?.();
    };
  }, []);
  ```

### 🟠 M2. `useAtollPhase` 定时器清理冲突
- **文件**: `src/useAtollPhase.ts:29-33, 35, 68`
- **问题**: 两个 `useEffect` 共享同一个 `timersRef`，一个 effect 的 cleanup 调用 `clearTimers` 会取消另一个 effect 设置的定时器。
- **修复**: 将 enter→loop 定时器与 transition 定时器使用独立的 ref 管理。

### 🟠 M3. `useAtollPhase` 多余依赖项导致不必要重跑
- **文件**: `src/useAtollPhase.ts:33`
- **问题**: `[phase, renderAct]` 依赖中 `renderAct` 未在 effect 体内使用，导致 `renderAct` 变化时不必要地重置定时器。
- **修复**: 从依赖数组中移除 `renderAct`。

### 🟡 M4. Rust 端内存集合无上限增长（复核：部分真实）
- **文件**: `src-tauri/src/lib.rs:274`
- **问题**: 原报告称 `requests`、`subagents`、`sessions` 等 Vec 无界增长，长时间运行可能导致内存耗尽。
- **复核结果**: 该描述过宽。`requests` 与 `known_sessions` 有 retention/auto-archive 清理逻辑；`session_token_usage` 与 `session_agent_map` 会保留到日切以维持 daily total；`active_subagents` 没有明显硬上限，归档后也只是标记 archived。
- **修复**: 为 `active_subagents`、hook waiters、token/session map 这类集合设置容量/生命周期策略；避免影响当日 token 统计。

## 四、逻辑错误与边界情况（Logic Errors）

### 🟠 L1. `isHookReady` 对 `scriptPath` 的判断过于宽松
- **文件**: `src/hookHealth.ts:28-31`
- **问题**: 任意 truthy 的 `scriptPath`（甚至是过期路径）都让 hook 状态为 "ready"，即使 `scriptFound` 显式为 `false`。
- **修复**: 要求 `scriptFound === true`，不应依赖 `scriptPath` 的 truthiness。

### 🟡 L2. `nodeFound` 合并逻辑将 "未知" 升级为 "已找到"（复核：需降级）
- **文件**: `src/hookHealth.ts:74`
- **问题**: 当两个值都是 `undefined` 时，`a.nodeFound !== false && b.nodeFound !== false` 求值为 `true`，静默将 "未知" 状态视为 "已找到"。
- **复核结果**: 前端 `normalizeHookStatus()` 和 Rust `HookStatus` serde 默认都把缺省 `nodeFound` 视为 true，因此这更像是当前数据模型的显式设计，而不是单点 bug。
- **修复**: 如果产品希望区分“未知”和“已找到”，应把 `nodeFound` 改成明确三态或强制后端总是返回实际检测结果；否则不应列为高优先级 bug。

### 🟡 L3. Demo 模式时间戳冻结
- **文件**: `src/demoSnapshot.ts:37`
- **问题**: `requestedAt: new Date().toISOString()` 在模块加载时求值一次，所有 demo 请求显示相同时间。
- **修复**: 改为在函数调用时动态生成时间戳。

### 🟡 L4. `toLocaleString()` 导致 Odometer 宽度估算跨 locale 不一致
- **文件**: `src/tokenCounterFormat.ts:11, 81-89`
- **问题**: `toLocaleString()` 的输出因 locale 不同而异（如阿拉伯数字、不间断空格），导致宽度估算偏差。
- **修复**: 使用固定 locale（`"en-US"`）或自定义格式化。

### 🟡 L5. Demo 模式缺少 Cursor Hook 状态回退
- **文件**: `src/tauri.ts:455-461` 与 `src/demoSnapshot.ts`
- **问题**: Claude 和 Codex 的 demo 回退已实现，但 Cursor 的 `getCursorHookStatus()` 缺少对应 demo 回退逻辑。
- **修复**: 补充 Cursor 的 demo 回退实现。

### 🟡 L6. `localRequests` 直接暴露引用
- **文件**: `src/tauri.ts:101, 118`
- **问题**: `getSnapshot()` 返回 `recent: localRequests` 直接引用，外部修改会破坏内部状态。
- **修复**: 返回 `[...localRequests]` 拷贝。

## 五、Rust 后端问题（Rust Backend）

### 🟠 B1. 20+ Mutex 无文档化锁顺序
- **文件**: `src-tauri/src/lib.rs:273-316`
- **问题**: 大量 Mutex 无明确的获取顺序约定，存在死锁风险。
- **修复**: 文档化锁获取顺序，考虑合并相关状态减少锁数量。

### 🟠 B2. Mutex 中毒级联崩溃
- **文件**: `src-tauri/src/lib.rs` 多处
- **问题**: `expect("state mutex poisoned")` — 任何线程持锁 panic 后，后续所有加锁尝试都会 panic，级联失败。
- **修复**: 使用 `lock().unwrap_or_else(|e| e.into_inner())` 容忍中毒锁，或改用 `parking_lot::Mutex`（不会中毒）。

### 🟡 B3. 遗留 Debug 日志
- **文件**: `src-tauri/src/debug_agent.rs:37-45`
- **问题**: 无条件写入 `~/.cursor/debug-d62394.log`，硬编码 session ID，疑似遗留调试代码。
- **修复**: 移除或改为条件编译（`#[cfg(debug_assertions)]`）。

### 🟡 B4. macOS 私有 API 启用
- **文件**: `src-tauri/tauri.conf.json:13`
- **问题**: `macOSPrivateApi: true` 用于 NSPanel 浮动效果，但可能在 macOS 更新时失效，且阻止 Mac App Store 分发。
- **修复**: 评估是否可改用公开 API 实现相同效果；若必须保留，记录风险。

## 验证方式

1. **安全修复验证**
   - CSP：构建后检查 webview 中 inline/script 行为是否符合预期。
   - Transcript 路径：添加单测覆盖合法 transcript、`..`、任意绝对路径、非 transcript 后缀。
   - Hook bridge：验证未带 token 的 localhost 请求被拒绝，合法 hook runner 请求可通过。
   - `openUrl()`：若做 defense-in-depth，补测 `https:`/`http:` 允许，`javascript:`/`file:` 拒绝；同时确认 Markdown 层默认过滤仍生效。

2. **内存泄漏验证**
   - 为 Tauri event listener 包装工具加单测：Promise resolve 前卸载时必须立即调用返回的 `UnlistenFn`。
   - 在 devtools / 日志中反复 mount/unmount App 与 TokenCounter，确认监听器数量不累积。

3. **前端测试基线**
   - 当前 `npm test -- --run` 已通过：11 个 test file、146 个 tests。
   - 仍需处理测试 stderr 中的 `validateDOMNesting` 警告：避免 `SessionListView` 中 `<button>` 嵌套 `<button>`。
   - 后续每轮修复后继续运行 `npm test -- --run`。

4. **类型与构建**
   - `npm run build`
   - `npx tsc --noEmit`（如单独保留）
   - Tauri release 相关修复后验证 `.github/workflows/release.yml` 中新增 test step。

5. **CI/发布验证**
   - 本地生成 `Fix-Atoll.command` 后确认首字节为 `#!`。
   - 对 `trigger-release.yml` 明确改用 PAT/GitHub App token 或删除该 workflow，避免继续误导。
